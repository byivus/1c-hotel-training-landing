## 2. КРИТИЧЕСКИЕ ПРОБЛЕМЫ (исправить первыми)

### 2.1 SSL-сертификат
- **Проблема:** Сайт работает без HTTPS. Google и AI-поисковики понижают HTTP-сайты в выдаче.
- **Действие:** Включить Enforce HTTPS в настройках GitHub Pages → Settings → Pages.

### 2.2 Несогласованность URL (canonical, sitemap, OG)
- **Проблема:** CNAME указывает на `ta1chotel.ru`, но canonical, sitemap, og:url и Schema.org ссылаются на `byivus.github.io/1c-hotel-training-landing/`. AI-поисковики получают противоречивые сигналы и могут не индексировать страницу.
- **Действие в index.html:**
  ```html
  <!-- БЫЛО -->
  <link rel="canonical" href="https://byivus.github.io/1c-hotel-training-landing/" />

  <!-- НУЖНО -->
  <link rel="canonical" href="https://ta1chotel.ru/" />
  ```
  То же самое для `og:url`, `twitter:url`, и `"url"` внутри Schema.org JSON-LD.

- **Действие в sitemap.xml:**
  ```xml
  <!-- БЫЛО -->
  <loc>https://byivus.github.io/1c-hotel-training-landing/</loc>

  <!-- НУЖНО -->
  <loc>https://ta1chotel.ru/</loc>
  ```

- **Действие в robots.txt:**
  ```
  Sitemap: https://ta1chotel.ru/sitemap.xml
  ```

- **Действие в llms.txt:**
  ```
  ## Canonical URL
  https://ta1chotel.ru/
  ```

### 2.3 Год в footer
- **Проблема:** `© 2024` — устаревший год.
- **Действие:** Заменить на `© 2024–2026` или сделать динамическим через JS.

## 4. ОПТИМИЗАЦИЯ ПОД AI-ПОИСК (AEO)

### 4.1 llms.txt — текущие проблемы

**Текущий файл:**
```markdown
# 1С:Отель Training Landing — TASolution

## What this site is
A landing page for an online training course...

## Course overview
- 3-day online course, 6 hours per day (18 hours total)
...
```

**Проблемы:**
1. ❌ Написан на **английском** — сайт русскоязычный, AI-поисковик получает несогласованный сигнал
2. ❌ Неверные данные: написано "6 hours per day (18 hours total)" — на самом деле 2 часа × 3 дня = 6 часов итого
3. ❌ URL ведёт на byivus.github.io, а не на ta1chotel.ru
4. ❌ Нет ссылок на конкретные секции или дополнительные ресурсы
5. ⚠️ Формат не соответствует спецификации llmstxt.org (нет blockquote с кратким описанием)

AI-поисковики (ChatGPT Search, Perplexity, Google AI Overview, Яндекс Нейро) предпочитают страницы с:

1. **FAQ-секцией на странице** (не только в Schema.org, но и визуально)
2. **Answer-first контентом** — прямые ответы на вопросы пользователей
3. **Определениями** — "Что такое 1С:Отель?"
4. **Числами и фактами** — конкретика вместо общих фраз
5. **E-E-A-T сигналами** — кто ведёт обучение, сколько лет опыта, сколько отелей обслужено

### 5.2 Проблемы

| Проблема | Приоритет | Рекомендация |
|----------|-----------|-------------|
| Нет ни одного изображения | Высокий | Добавить: фото процесса обучения, скриншоты 1С:Отель, фото преподавателя |
| Emoji вместо иконок | Средний | Заменить на SVG-иконки (Lucide/Heroicons) — профессиональнее и стабильнее отображение |
| Нет социального доказательства | Высокий | Добавить: отзывы, логотипы клиентов, счётчик обученных |
| Нет цены | Средний | Хотя бы "от X руб." или "индивидуальный расчёт" |
| CTA ведёт на внешний сайт | Низкий | Рассмотреть встроенную форму заявки |
| Нет видео | Средний | Добавить короткое демо-видео (1-2 мин) |

| Мобильное меню не показывает CTA | Средний | При открытии бургера CTA скрыт (display:none при <720px) |
| Нет breadcrumbs | Низкий | Для одностраничника не критично |

### 5.3 Доступность (a11y)

| Проблема | Действие |
|----------|----------|
| `aria-label` только на бургере и scroll-top | Добавить `role="navigation"` на nav, `aria-expanded` на аккордеон |
| Аккордеон без клавиатурной навигации | Добавить `role="button"`, `tabindex="0"`, обработку Enter/Space |
| Контраст текста hero | Проверить — белый на синем может быть OK, но полупрозрачный текст (opacity: .65-.88) может не пройти WCAG AA |
| Нет `skip-to-content` ссылки | Добавить для screen readers |
| Emoji как иконки | Не все screen readers корректно читают emoji. Добавить `aria-hidden="true"` или заменить на SVG с `alt` |
